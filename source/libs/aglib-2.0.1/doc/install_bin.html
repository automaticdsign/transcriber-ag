INCOMPLETE: Installing binary packages

Windows zip package

A set of AGLIB components compiled on an Intel Windows platform are distributed as a zip package. This package includes all components of AGLIB described in the Components of AGLIB and required third-party packages. However, a Python interpreter, a Tcl/Tk interpreter and a Java Virtual Machine are not included in the package, and they need to be installed separately if any of these languages are to be used.

When unpacked, the zip file installs the following files: libag.dll, ag\agfio_plugin_*.dll, ag\java\*, ag\python\*, and ag\tcl\*.

The libag.dll is the main .dll file that can be used from C and C++. The ag\agfio_plugin_*.dll files are the optional plugin .dll files for the file I/O architecture of AGLIB.

We advise against installing the .dll files in a system directory. Instead, we recommend setting a path to include the .dll files from a start-up batch file for each application.

Please note that there are two ag.dll files in the pacakge. One is for Tcl and one is for Python, and they are incompatible with each other.

If the zip file is unpacked in the C:\aglib2.0 directory, the following DOS commands will set the environment variables to use the libraries of the package. 

C:\> set Path=C:\aglib2.0;C:\aglib2.0\ag;\aglib2.0\ag\tcl;%PATH%
C:\> set TCLLIBPATH=C:\aglib2.0\ag\tcl;%TCLLIBPATH%


C:\> wish
% package require ag
